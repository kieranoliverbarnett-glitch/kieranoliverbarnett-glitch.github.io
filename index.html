Pillar 1: Team Enablement & AI Support Tools

Live Team Tool Prototype Streamlit + Claude API
Agent Empowerment

Context-Aware Insurance Knowledge Base

To demonstrate how to scale support capacity without inflating headcount, I built a customer-facing chatbot prototype designed to handle complex life insurance inquiries. Connecting context from a general knowledge base directly to the Claude API, it delivers highly accurate answers to common questions. To protect the customer experience, the system features built-in safety routing: the moment an inquiry falls outside the documentation scope, it seamlessly guides the user to connect with a live representative. Note: All information contained in the knowledge base consists entirely of synthetic data compiled from public sources for testing purposes only, and does not contain any proprietary or company-specific information.

💡 Operational Cross-Over: While built for customer self-service, this exact architecture can easily be deployed internally to replace traditional, clunky internal knowledge bases—helping front-line agents instantly surface accurate answers in highly complex, documentation-heavy compliance environments.
